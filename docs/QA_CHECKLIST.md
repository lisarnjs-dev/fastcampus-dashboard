# QA 테스트 체크리스트

배포 환경에서 처음부터 끝까지 어드민 → 수강생 순서로 테스트하는 시나리오입니다.
각 항목 앞에 `[ ]`을 `[x]`로 바꾸면서 진행하세요.

---

## 사전 준비

- 브라우저 시크릿 창(또는 다른 브라우저) 두 개 준비
  - 창 A: 어드민 테스트용
  - 창 B: 수강생 테스트용
- 테스트에 쓸 더미 CSV 파일 준비 (아래 형식)

```csv
name,email,dashboard_group
홍길동,test1@example.com,A
김영희,test2@example.com,A
이철수,test3@example.com,B
```

## 1단계 — 어드민 인증

### 1-1. 비로그인 접근 차단

| # | 테스트 항목 | 기대 결과 |
|---|-------------|-----------|
| □ | `/admin` 직접 접속 | `/admin/login`으로 리다이렉트 |
| □ | `/admin/cohorts` 직접 접속 | `/admin/login`으로 리다이렉트 |
| □ | `/admin/students` 직접 접속 | `/admin/login`으로 리다이렉트 |
| □ | `/admin/attendance` 직접 접속 | `/admin/login`으로 리다이렉트 |
| □ | `/admin/missions` 직접 접속 | `/admin/login`으로 리다이렉트 |

### 1-2. 로그인 폼

| # | 테스트 항목 | 기대 결과 |
|---|-------------|-----------|
| □ | 비밀번호 없이 제출 | 제출 안 됨 (HTML required) |
| □ | 잘못된 비밀번호 입력 | 에러 메시지 표시 |
| □ | 올바른 비밀번호 입력 | `/admin` 대시보드로 이동 |
| □ | 로그인 후 `/admin/login` 재접속 | 이미 로그인 상태면 `/admin`으로 이동 또는 로그인 폼 표시 |
